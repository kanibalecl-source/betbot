BETBOT SERVER PATCH v9.7 - VOLLEYBALL WALK-FORWARD CHAMPION-CHALLENGER

REQUIRES
- Successfully deployed and verified v9.6.
- Volleyball remains shadow-only.
- The isolated volleyball SQLite database is on the persistent Railway volume.

PURPOSE
- Validate every reproducible volleyball candidate on later, unseen matches.
- Compare the challenger with the fixed current Elo champion.
- Measure out-of-sample Brier score, log loss and calibration.
- Store every validation as immutable evidence for v9.8 stability controls.

SAFETY CONTRACT
- Football source, history, models, databases and execution are unchanged.
- Existing volleyball schema is migrated additively; no table or row is deleted.
- A validation never changes the active model.
- Automatic promotion and real execution remain disabled.
- A positive result still requires manual approval.
- With fewer than 100 eligible games, v9.7 remains in the expected
  WAITING_REPRODUCIBLE_CANDIDATE state.

VALIDATION CONTRACT
- Expanding-window chronological folds.
- Default minimum: 40 earlier training games, 20 later test games, 3 folds.
- Matches sharing a timestamp are never split across train and test.
- Every probability is emitted before that match result updates either model.
- Candidate and champion are scored on exactly the same observations.
- Paired deterministic bootstrap confidence intervals are calculated for
  Brier and log-loss improvement.
- Positive validation requires:
  * enough out-of-sample folds,
  * strict chronological separation,
  * positive lower 95% CI for Brier improvement,
  * positive lower 95% CI for log-loss improvement,
  * calibration not worse than the champion,
  * a reproducible candidate.
