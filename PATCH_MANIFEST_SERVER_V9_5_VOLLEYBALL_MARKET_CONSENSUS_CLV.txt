BETBOT SERVER PATCH v9.5 - VOLLEYBALL MARKET CONSENSUS AND CLV

REQUIRES
- Successfully deployed and verified v9.4.
- Volleyball remains shadow-only.
- Existing API-Sports volleyball key and v9.x variables remain configured.

PURPOSE
- Replace dependence on a single bookmaker quote with a robust multi-book market
  reference.
- Remove bookmaker margin before calculating the market consensus probability.
- Measure prediction quality against the last verified market snapshot before
  match start (closing line value, CLV).

SAFETY CONTRACT
- Football source, history, models, databases and execution are unchanged.
- Existing volleyball schema is migrated additively; no table or row is deleted.
- Existing v9.0-v9.4 picks, features and settlement evidence are not rewritten.
- Consensus snapshots, closing snapshots, pick links and CLV are immutable.
- Training, model promotion and real execution remain disabled.

CHANGES
- HOME/AWAY quotes are paired per bookmaker and only complete valid pairs enter
  the consensus.
- Every pair is de-vigged independently; the consensus uses the robust median
  probability across bookmakers.
- Quotes above 100.0 and pairs with implausible overround are rejected.
- New picks require at least two valid bookmakers by default.
- The executable best quote is selected only from bookmakers admitted to the
  validated consensus.
- Odds refresh becomes more frequent as match start approaches.
- Every new pick is linked to the exact market-consensus snapshot used at issue.
- The closing snapshot is the latest consensus strictly before scheduled start;
  post-start prices can never become the close.
- CLV is recorded once per settled pick against both best closing price and
  no-vig closing fair price.
- Health output reports consensus, insufficient-book and CLV counters.

OPTIONAL VARIABLE
- BETBOT_VOLLEYBALL_MIN_BOOKMAKERS=2
  Default is 2, so no Railway variable change is required.
