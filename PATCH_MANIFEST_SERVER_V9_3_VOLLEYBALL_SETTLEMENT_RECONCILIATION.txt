BETBOT SERVER PATCH v9.3 - VOLLEYBALL SETTLEMENT & RECONCILIATION

REQUIRES
- Successfully deployed and verified v9.2.
- Volleyball remains shadow-only.

PURPOSE
- Settle volleyball shadow picks automatically from provider results.
- Keep unresolved historical picks inside the result-fetch window.
- Detect provider/result changes without rewriting prior evidence.

SAFETY CONTRACT
- Football source, history, models and execution are unchanged.
- Existing volleyball schema is migrated additively; no table or row is deleted.
- Pick closure is idempotent and immutable settlement evidence is append-only.
- A changed or contradictory result is recorded as a visible MISMATCH.
- Training, model promotion and real execution remain disabled.

CHANGES
- Every cycle fetches yesterday, today and tomorrow.
- Dates of all still-open picks are added automatically to the fetch window.
- An OPEN pick can be closed only once.
- Settlement evidence is written only after a successful first closure.
- Closed picks are recalculated against current provider evidence.
- CONSISTENT and MISMATCH observations are stored in an append-only audit.
- Health output reports settlement audit and mismatch counts.
